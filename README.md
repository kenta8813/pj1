プロジェクト要件まとめ
プロジェクト概要
LLMを活用したデータキュレーション型ポータルサイトの構築。
汎用基盤として設計し、初回ユースケースは日本の自治体・子育て支援情報の保護者向けポータル。

技術スタック（モノレポ構成）



|レイヤー     |技術                            |
|---------|------------------------------|
|データ収集    |Laravel 13 + AI SDK           |
|データ保管    |JSONストア or SQLite（初期）         |
|フロントエンド  |Next.js + React + Tailwind CSS|
|LLMプロバイダー|OpenRouter（メイン）+ Ollama（ローカル） |

主要モジュール
データ収集モジュール
	•	入力： エントリーポイントURL・収集指示・出力テンプレート（JSON）
	•	動作： エントリーポイントからリンクを再帰的に辿り、LLMでデータ抽出
	•	出力： テンプレート形式のJSON
フロントエンド
	•	SSR / SSG構成
	•	SEO・AdSense対策を最重要視

マネタイズ
	•	**インプレッション型広告（AdSenseなど）**を前提
	•	SEO・AdSense対策は極めて重要な非機能要件

開発環境・ハーネス
Claude Code設定方針
	•	承認要求を最小化（settings.jsonで設定）
	•	機密情報流出のみ厳密にブロック
	•	セッション間コンテキスト共有のためのメモリー保存機構
MCP連携
	•	laravel-boost MCP
	•	next-dev-tool MCP
サブエージェント構成



|エージェント     |役割                                    |
|-----------|--------------------------------------|
|パーソナルエージェント|メインセッション担当・ユーザー対話・ハーネス改善・他エージェントへの振り分け|
|ディレクター     |企画・デザイン・仕様・要件・ユーザーストーリー               |
|リードエンジニア   |技術、実装プランニング                              |
